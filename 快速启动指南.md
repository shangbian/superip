# 快速启动指南

## 🎯 5分钟快速启动

### 步骤1：配置环境变量（首次必做）

```bash
# 进入后端目录
cd backend

# 将 config.txt 重命名为 .env
ren config.txt .env
```

### 步骤2：安装依赖（首次必做）

```bash
# 在backend目录下
npm install
```

### 步骤3：启动后端

**方式一：使用启动脚本（推荐）**
```bash
双击 启动后端.bat
```

**方式二：手动启动**
```bash
cd backend
npm run dev
```

后端服务将在 http://localhost:3000 启动

### 步骤4：启动前端

**方式一：使用Live Server（最推荐）**
1. 在VS Code中安装Live Server插件
2. 右键点击 `frontend/index.html`
3. 选择 "Open with Live Server"
4. 自动在浏览器打开

**方式二：使用启动脚本**
```bash
双击 启动前端.bat
```

**方式三：使用Python**
```bash
cd frontend
python -m http.server 8080
```
然后在浏览器访问 http://localhost:8080

## ✅ 验证启动

### 1. 检查后端
访问：http://localhost:3000/health

应该看到：
```json
{
  "status": "healthy",
  "timestamp": "...",
  "cozeToken": "已配置",
  "workflowId": "7599232922009583626"
}
```

### 2. 检查前端
访问：http://localhost:8080（或Live Server的端口）

应该看到四大分类和36个智能体卡片

## 🎮 开始使用

1. 在首页选择一个分类下的智能体
2. 阅读智能体的介绍和输入要求
3. 在输入框填写相关信息
4. 点击"生成内容"按钮
5. 等待AI生成结果
6. 点击"一键复制"复制生成的内容

## 🐛 遇到问题？

### 问题1：后端启动失败
```bash
# 清除依赖重新安装
cd backend
rm -rf node_modules package-lock.json
npm install
```

### 问题2：前端无法访问后端
- 检查后端是否正常启动（访问health接口）
- 检查 `.env` 中的 `CORS_ORIGIN` 是否与前端地址一致
- 检查防火墙是否阻止了3000端口

### 问题3：API返回错误
- 查看后端控制台日志
- 查看 `logs/` 目录中的日志文件
- 确认Token是否正确配置

### 问题4：生成内容为空
- 检查扣子平台工作流是否正常运行
- 查看后端日志中的API响应
- 确认choice参数是否正确传递

## 📝 常用命令

```bash
# 开发模式启动后端（支持热重载）
cd backend
npm run dev

# 生产模式启动后端
cd backend
npm run build
npm start

# 查看后端日志
cat logs/info.log      # 常规日志
cat logs/error.log     # 错误日志

# 重新安装依赖
cd backend
npm install

# 更新Token
# 编辑 backend/.env 文件中的 COZE_TOKEN
```
