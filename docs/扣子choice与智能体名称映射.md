# 扣子工作流 `choice` 与外部智能体名称映射

本文档与前端 `多智能体/frontend/app.js` 中 **`resolveWorkflowChoice`**、**`WORKFLOW_CHOICE_OVERRIDE`** 保持一致，便于在扣子画布（选择器 / 分支编号）与产品侧智能体名称之间对齐。

## 说明

1. **后端请求参数**：调用 `/api/coze/workflow/execute` 时，请求体中的 **`choice`** 由前端 `resolveWorkflowChoice(当前智能体 id)` 计算得出，并传入扣子工作流。
2. **默认规则**：绝大多数情况下 **`choice` 与前端智能体 `id` 相同（1～36）**。
3. **例外（必须与工作流选择器一致）**：历史上为避免与「自我介绍 / 痛点」分支冲突，对两个智能体做了 **互换**（见下表「互换」列）。

```text
WORKFLOW_CHOICE_OVERRIDE = { 5: 9, 9: 5 }
```

含义：

| 用户在前端选中的智能体 | 前端 `id` | 实际传入扣子的 `choice` |
|------------------------|-----------|-------------------------|
| 大字报（自我介绍）     | **5**     | **9**                   |
| 痛点共鸣               | **9**     | **5**                   |

4. **「节点数字」**：若扣子画布里分支标注为 **1～36**，一般即下表中的 **`choice`**；若画布上写的是「调用 LLM N」或自定义节点名，请以 **工作流调试里收到的 `parameters.choice`** 为准，并与下表核对。

- 若在扣子侧 **新增/调整分支编号**，需同步修改前端 `WORKFLOW_CHOICE_OVERRIDE` 与 `allAgentsMap`，并更新本文件。
