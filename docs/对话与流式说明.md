# 用户与智能体对话 · 说明

## 对话标题与展示

- 进入某个智能体后，标题为：**用户与「某某智能体」的对话**。
- 副标题说明：支持流式更新对话气泡；同一会话使用 `conversation_id` 保留上下文。

## 提示气泡（快捷输入）

- 下方展示的提示词气泡来自该智能体的「输入要求」示例。
- **点击气泡**：会**直接以该文案发起一次对话**（不填入输入框），即点即发。
- 若希望只填入输入框再自己编辑，可改为在输入框内输入或粘贴内容后点「发送」。

## conversation_id 与多轮上下文

- **首次调用**：不传 `conversation_id`，由 Coze API 创建新会话并在响应中返回 `conversation_id`。
- **后续调用**：前端将该智能体的 `conversation_id` 随请求传给后端，后端再传给 Coze，由 API 自动保留历史消息上下文。
- **choice**：通过 `parameters` 传递，同一智能体的多轮对话中保持不变。
- **清除记录**：用户点击「清空对话」时，会清除当前智能体的对话记录**以及**该智能体对应的 `conversation_id`，下次发送将重新创建新会话。

## 流式输出

- **当前实现**：后端一次性拿到 Coze 工作流完整结果后，前端以**打字机效果**逐字展示，并支持「停止响应」立即展示全文。
- **若需真实流式**：需 Coze 工作流接口支持流式返回（如 SSE），并由后端将流式响应转发给前端，前端按 chunk 实时更新对话气泡。
